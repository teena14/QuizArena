# AI Quiz Generation Returns 502 Bad Gateway

## Details
- **Date**: 2026-07-18
- **Severity**: 🔴 Critical
- **File**: `src/app/api/teacher/quizzes/generate/route.ts`
- **Issue**: When a teacher attempted to generate a quiz using the "AI Question Generator" feature (by topic or file upload), the API route `/api/teacher/quizzes/generate` returned a `502 Bad Gateway` error instead of the JSON response.

## Root Cause
The `502 Bad Gateway` error generated by the Next.js Dev Server proxy was caused by two main issues related to the new `@google/genai` SDK:

1. **Top-level Environment Variable Evaluation**: The SDK was initialized at the top level of the file (`const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });`) outside of the Next.js `POST` function. Depending on the environment and how Next.js hydrates environment variables, `process.env.GEMINI_API_KEY` can be `undefined` during module load time, causing the initialization to throw an error and crash the route load process.
2. **Bundling Enums in Next.js**: The code imported and used `Type` and `Schema` from `@google/genai` to build the `responseSchema`. Because of how Next.js bundles server-side code (using Webpack or Turbopack), importing these Enums sometimes fails resolution or crashes the bundler, especially in specific edge/node hybrid runtimes or strict module boundary settings.

## Fix Applied
To resolve the issue and make the integration robust:
1. **Lazy Initialization**: Moved the `GoogleGenAI` initialization *inside* the `POST` function. This ensures that `process.env.GEMINI_API_KEY` is fully populated when the code actually runs during a request.
2. **Literal String Schema Definitions**: Replaced the `Type.ARRAY` and `Type.OBJECT` enum references with their literal string equivalents (`"ARRAY"`, `"OBJECT"`). The `@google/genai` SDK natively supports string literal types, which bypasses the need to import runtime enums and eliminates Next.js bundler compatibility issues.

```typescript
// Fixed Code Snippet
import { GoogleGenAI } from "@google/genai";

const responseSchema = {
  type: "ARRAY",
  items: {
    type: "OBJECT",
    properties: {
      text: { type: "STRING" },
      // ...
    },
    // ...
  },
};

export async function POST(req: NextRequest) {
  try {
    const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });
    // ...
  }
}
```
