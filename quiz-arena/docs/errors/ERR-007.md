# ERR-007: Refactor TypeScript Errors

## Description
During the refactoring phase, several TypeScript errors were introduced when moving inline types to shared types.
Specifically, `isPublished` was required by the new shared `Quiz` type, but was not being queried from the database using Prisma `select`. Additionally, `q.attempts?.length` operations caused `Object is possibly 'undefined'` errors.

## Context
Refactoring components to use shared interfaces from `src/types/index.ts`.

## Solution
1. Added `isPublished: true` to all Prisma `select` queries fetching Quizzes (e.g., in `src/app/(dashboard)/student/...`).
2. Updated optional chaining logic to use Nullish Coalescing for length checks where required: `(q.attempts?.length ?? 0) > 0`.
